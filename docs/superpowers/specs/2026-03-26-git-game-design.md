# GitQuest — Интерактивная веб-игра для изучения Git и GitHub

## Обзор

Веб-игра, обучающая системе контроля версий Git и платформе GitHub через интерактивные визуализации, квизы и практические челленджи. Игрок проходит путь от новичка до эксперта в нарративной обёртке с геймификацией.

## Целевая аудитория

- Начинающие разработчики, которые уже пишут код, но плохо знают Git
- Студенты университетов/курсов, осваивающие Git как часть программы
- Широкий диапазон уровней — от базовых до продвинутых тем

Предполагается, что пользователь умеет пользоваться терминалом и писать код.

## Стек технологий

- **Framework:** Next.js (App Router) + React + TypeScript
- **Визуализация:** reactflow (git-графы) + framer-motion (анимации)
- **Стилизация:** Tailwind CSS
- **Авторизация:** next-auth (GitHub OAuth)
- **Хранение прогресса:** localStorage (по умолчанию) + Vercel KV (после входа через GitHub)
- **Деплой:** Vercel

## Концепция взаимодействия

Игра — интерактивный визуальный учебник:
- Анимированные визуализации: показываем как команды влияют на git-граф
- Имитация терминала: анимированная демонстрация ввода команд и вывода
- Квизы для проверки понимания
- Челленджи — практические задания для выполнения на собственном компьютере игрока

Реальный git на сервере или в браузере не выполняется. Игрок практикуется на своей машине.

## Архитектура: Component-driven + File-based Content

Каждый уровень — отдельная папка с конфигом и React-компонентами. Git-визуализация — единый переиспользуемый движок. Уровни регистрируются автоматически.

## Система уровней

### Структура уровня

Каждый уровень определяется файлом `meta.json`:

```json
{
  "id": "01-init",
  "title": "Создаём первый репозиторий",
  "description": "Узнай что такое git init и зачем он нужен",
  "difficulty": 1,
  "xp": 100,
  "section": "basics",
  "tags": ["init", "repository"],
  "prerequisites": [],
  "estimatedMinutes": 5
}
```

### 7 разделов

| Раздел | Сюжет | Уровней | Примеры команд |
|--------|-------|---------|----------------|
| Основы | "Первый день: настраиваем рабочее место" | 5-6 | init, add, commit, status, log, diff |
| Ветвление | "Первая задача: работаем над фичей" | 5-6 | branch, switch, merge, rebase, HEAD |
| Удалённые репо | "Подключаемся к команде" | 4-5 | clone, remote, push, pull, fetch |
| GitHub | "Первый Pull Request" | 4-5 | PR, Issues, Fork, Code Review |
| Продвинутое | "Что-то пошло не так — спасаем ситуацию" | 5-6 | stash, cherry-pick, reset, revert, reflog |
| Командная работа | "Релиз: работаем вместе" | 4-5 | merge conflicts, Git Flow, trunk-based |
| Git internals | "Бонус: заглядываем под капот" | 3-4 | objects, blobs, trees, refs, HEAD |

**Итого: ~30-37 уровней**

### Шаги внутри уровня

1. **Нарратив** — персонаж-наставник объясняет контекст
2. **Визуализация** — анимированная демонстрация работы команды
3. **Квиз** — проверка понимания (1-3 вопроса)
4. **Челлендж** — практическое задание для своего компьютера

### Навигация

- Свободная — все уровни открыты
- Рекомендованный порядок прохождения
- Индикаторы сложности: зелёный (1-2), жёлтый (3-4), красный (5)

## Git Visualizer — движок визуализации

Единый переиспользуемый компонент для всех уровней.

### Что визуализируем
- Коммиты как узлы графа (хеш, сообщение, автор)
- Ветки как цветные линии
- HEAD — указатель на текущую позицию
- Merge — слияние двух линий
- Rebase — перемещение коммитов
- Staging area, working directory

### Технологии
- `reactflow` — рендер графа (узлы + рёбра)
- `framer-motion` — анимации переходов

### Декларативный API

Уровень передаёт начальное состояние и шаги — визуализатор анимирует переходы:

```tsx
<GitVisualizer
  initial={[
    { id: "a1", message: "Initial commit", branch: "main" },
    { id: "a2", message: "Add README", branch: "main", parent: "a1" },
  ]}
  steps={[
    {
      command: "git branch feature",
      add: [{ id: "a2", branch: "feature", ref: true }],
      highlight: "feature",
      narration: "Новая ветка создана — она указывает на тот же коммит"
    },
    {
      command: "git switch feature",
      moveHead: "feature",
      narration: "HEAD теперь на ветке feature"
    }
  ]}
/>
```

## Геймификация и нарратив

### Персонаж-наставник
- Имя: Алекс — опытный разработчик
- Встречает игрока "в первый день на работе"
- Появляется в диалоговом пузыре на каждом уровне
- Даёт контекст, подсказки, реагирует на ответы квиза
- Тон: дружелюбный, неформальный, с юмором

### Система XP и рангов
- XP за прохождение уровней: 100-300 в зависимости от сложности
- Бонусный XP за правильные ответы квиза с первой попытки
- Ранги: Intern → Junior → Middle → Senior → Lead → Architect

### Бейджи (достижения)
- "Первый коммит" — пройти первый уровень
- "Ветвление мастер" — пройти все уровни по веткам
- "Конфликт решён" — пройти уровень по merge conflicts
- "Git археолог" — пройти Git internals
- "Speedrun" — пройти уровень меньше чем за 2 минуты
- "Перфекционист" — все квизы без ошибок в разделе

## Авторизация и хранение данных

### Без регистрации (по умолчанию)
Прогресс в `localStorage`:

```json
{
  "player": {
    "rank": "Junior",
    "totalXp": 850,
    "completedLevels": ["01-init", "02-first-commit", "03-staging"],
    "quizScores": { "01-init": { "correct": 3, "total": 3 } },
    "badges": ["first-commit", "speedrun"],
    "currentSection": "branching"
  }
}
```

### GitHub OAuth (опционально)
- Кнопка "Войти через GitHub" в профиле
- Next.js API Route `/api/auth/github` для обмена code → token
- После входа прогресс из localStorage синхронизируется в Vercel KV
- Конфликт данных: берём прогресс с большим XP

### Почему Vercel KV
- Бесплатный tier достаточен
- Key-value модель подходит: один JSON-документ на пользователя
- Нативная интеграция с Next.js на Vercel

## UI и навигация

### 4 основных экрана

1. **Главная — Карта уровней**
   - Разделы с прогресс-барами
   - Карточки уровней с индикаторами (пройдено / текущий / не пройдено)
   - Навбар: логотип, XP, ранг, кнопка входа

2. **Страница уровня**
   - Левая панель: нарратив (наставник) → шаги → имитация терминала
   - Правая панель: git-визуализатор
   - Мобильная версия: контент сверху, визуализация снизу

3. **Квиз (внутри уровня)**
   - Вопрос + 4 варианта ответа
   - Мгновенная обратная связь: правильно/неправильно + объяснение
   - Бонусный XP за первую попытку

4. **Профиль игрока**
   - Аватар, ранг, XP до следующего ранга
   - Статистика: уровней пройдено, точность квизов, бейджи
   - Коллекция бейджей (открытые + заблокированные)
